# VM XML 配置生成器 - 改进计划

> 基于代码审查，按优先级排序。✅ 已完成 / 🔲 待实施

---

## 一、高优先级（稳定性 / 正确性）

### 1.1 ✅ 消除 `unwrap()` 崩溃风险

| 文件 | 修复方式 |
|------|----------|
| `src/main.rs` | `load_icon` 改为 `unwrap_or_else`；字体加载改为 `if let` |
| `src/panels/os_panel.rs` | `kernel.as_mut().unwrap()` 改为 `get_or_insert_with` |
| `src/panels/devices_panel.rs` | TPM 后端 5 处 `unwrap` 改为直接操作 `ref mut backend`；smartcard 改为 `if let` |
| `src/panels/advanced/hypervisor_features_panel.rs` | `as_ref().unwrap()` 改为 `as_deref().unwrap_or("off")` |

---

### 1.2 ✅ 完成引导设备上移/下移

`src/panels/os_panel.rs` 中每个引导设备行内联 ⬆️/⬇️ 按钮，使用 `boot_list.swap(i, i±1)` 实现。

---

### 1.3 ✅ 添加输入验证

新增 `src/model/validation.rs`，提供以下验证函数：
- `validate_uuid` — UUID 格式（8-4-4-4-12 十六进制）
- `validate_mac` — MAC 地址格式（xx:xx:xx:xx:xx:xx）
- `validate_memory_size` — 内存大小（正整数）
- `validate_vcpu_count` — vCPU 数量（1-1024）
- `validate_path` — 文件路径（不含空字符）

UI 层已在 UUID（`general_panel.rs`）和 MAC 地址（`devices_panel.rs`）输入框旁显示 ⚠ 红色提示。

---

### 1.4 ✅ 修复 MAC 地址生成随机性不足

`devices_panel.rs` 中时间戳方案改为 `uuid::Uuid::new_v4()` 随机字节，利用已有依赖，无需新增 crate。

---

## 二、中优先级（代码质量 / 可维护性）

已提取通用宏到 `panels/macros.rs`：

- `field_row_with_validation!` — 带验证提示的字段
- `field_row_opt!` — Option<String> 类型字段

- `combo_row!` / `combo_row_str!` — 下拉框
- `slider_row!` / `slider_row_simple!` — 滑块
- `checkbox_row!` — 复选框

已在 `smbios_panel.rs` 和 `memory_tuning_panel.rs` 中应用宏。

---

### 2.2 ✅ 减少 XML 生成代码重复

已在 `xml_gen/mod.rs` 中提取辅助函数：
- `write_element` — 写入基础元素
- `write_element_with_attrs` — 写入带属性的元素
- `write_empty_element` — 写入自闭合元素

---

### 2.3 ✅ 改进状态管理

- `status_message` 类型加入 `std::time::Instant` 时间戳
- 新增 `set_status()` 辅助方法统一设置消息
- `show_status_bar` 中超过 3 秒自动清除
- `update()` 中消息存在时调用 `request_repaint_after` 确保定时刷新

---

### 2.4 ✅ 定义自定义错误类型

新增 `src/error.rs`，提供以下错误类型：
- `AppError::XmlGeneration` — XML 生成错误
- `AppError::FileOperation` — 文件操作错误（From std::io::Error）
- `AppError::Validation` — 配置验证错误
- `AppError::JsonError` — JSON 序列化/反序列化错误
- `AppError::XmlParseError` — XML 解析错误
- `AppError::UiError` — UI 错误

已在 `Cargo.toml` 添加 `thiserror = "1.0"`。

---

### 2.5 ✅ 消除不必要的克隆

`general_panel.rs` 中 UUID、描述、标题字段的多余 `.clone()` 已消除，改为直接操作 `get_or_insert_with` 返回的可变引用。

---

## 三、低优先级（功能增强）

### 3.1 ✅ 配置持久化（保存/加载 JSON）

已添加 `serde_json` 依赖，在 `app.rs` 中实现 `save_config` / `load_config`，文件菜单新增"保存配置..."和"加载配置..."入口，支持将 `VMConfig` 序列化为 JSON 文件并重新加载。

---

### 3.2 ⏸️ XML 导入功能

已创建 `src/xml_import.rs` 模块和 `import_from_xml` / `import_from_file` 函数，在文件菜单中添加了"导入 XML..."入口。

**当前状态**：基础框架已完成，但 `quick-xml` 的 serde 解析器无法直接解析 libvirt XML 的扁平结构到 VMConfig 的嵌套结构。需要手动编写解析逻辑来处理 `<domain>` 标签下的直接子元素（`<name>`, `<uuid>`, `<memory>`, `<vcpu>` 等）。

**后续工作**：需要为每个字段编写手动解析逻辑，或修改 VMConfig 模型添加自定义的 Deserialize 实现。

---

### 3.3 ✅ 撤销/重做

已实现历史记录机制：
- `history: Vec<VMConfig>` — 存储历史配置
- `history_index: usize` — 当前索引
- `push_history()` — 保存当前配置
- `undo()` / `redo()` — 撤销/重做
- 菜单项：↩ 撤销 (Ctrl+Z) / ↪ 重做 (Ctrl+Y)
- 自动限制历史记录数量为 50 条

---

### 3.4 ✅ 单元测试

新增 `src/xml_gen/tests.rs`，包含 11 个测试，全部通过：
- `test_generate_basic_vm` — 基础 XML 结构
- `test_domain_type_attribute` — domain type 属性
- `test_uuid_included_when_set` — UUID 有值时输出
- `test_uuid_omitted_when_none` — UUID 为 None 时不输出
- `test_format_xml_indentation` — 格式化缩进
- `test_serialization_roundtrip` — JSON 序列化往返
- `test_vcpu_count_in_xml` — vCPU 数量输出
- 4 个验证函数测试（uuid / mac / memory / vcpu）

---

### 3.5 ✅ XML 格式化改进

`format_xml()` 已改用 `quick-xml` 内置的 `Writer::new_with_indent` 进行格式化，正确处理 CDATA、注释、特殊字符转义和缩进。

---

### 3.6 ✅ 依赖优化

- ✅ 已移除 `serde-xml-rs`（如有），使用 `quick-xml` 作为唯一 XML 库
- ✅ 已用 `uuid` 替代时间戳 MAC 生成，无需额外引入 `rand`

---

### 3.7 ✅ Clippy 代码质量改进

已修复所有 Clippy 警告：
- 修复文档注释后的空行问题（`validation.rs`、`macros.rs`）
- 修复冗余模式匹配和无用转换（`xml_gen/mod.rs`）
- 修复可折叠 if 语句（`xml_import.rs`）
- 使用结构体简化复杂返回类型（`xml_import.rs`）
- 添加数字分隔符提高可读性（`cpu_tuning_panel.rs`）
